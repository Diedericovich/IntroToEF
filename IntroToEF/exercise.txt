Groepsopdracht EF
-----------------------------------
Maak of breid de bestaande Samurai - Console App uit zodat er data uit de database opgehaald en ingevoerd kan worden via de console.
Deze opdracht is grotendeels carte-blanche. Probeer zelf wat te spelen met het gebruik van een database in een applicatie.

Wat moet er zeker in de app zitten?
OK - Een gebruiker moet een samurai kunnen toevoegen via de console
OK - Er moet ook optionele gerelateerde data kunnen toegevoegd worden.
OK - Heeft een samurai paarden?
OK - Heeft hij in veldslagen gevochten?
OK - Er moet een detail overzicht van een samurai kunnen opgevraagd worden waarin alle gerelateerde data getoond wordt
OK - Er moet een filter mogelijkheid zijn. Toon bv enkel de samurai met paarden
OK - Toon een overzicht van alle samurai die in een bepaalde veldslag gevochten hebben
OK - De data van een bestaande samurai moet aangepast kunnen worden of verwijderd worden.
 
Wil je meer uitdaging? Probeer de functionaliteit om te draaien. 
Zo is er bijvoorbeeld de mogelijkheid om een samurai met paarden toe te voegen. 
Kan je ook paarden met samurai toevoegen? Kan je battles toevoegen met samurai die paarden en quotes hebben?

Extra uitdaging: Kan je extra tabellen verzinnen die te maken hebben met dit project? 
Denk bijvoorbeeld aan familieleden? 
Aan welke heer /daimyo leggen ze verantwoording af? Welke religie volgen ze etc... 
Voorzie ook de mogelijkheid om deze Entities op te slaan, aan te passen en te verwijderen.
Hoe je dit aanpakt en hoe je de data toont is volledig jullie beslissing. Veel success!
